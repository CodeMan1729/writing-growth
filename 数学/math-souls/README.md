## 核心设计理念

### 关于「外挂击杀」的设计

外挂击杀不是惩罚机制，是一个**诚实记录工具**。给0 XP是因为看答案没有真正建立理解，但这件事本身不该被羞耻化——有时候你确实需要看答案才能理解这道题在干什么。诅咒标记的意义是：下次看到这道题时，提醒自己「这里还有欠债」。

---

*最后更新：2026-03-20*
*当前版本：v0.1.0（本地单机原型）*
为什么是魂类游戏？

魂类游戏最聪明的地方在于：**把失败本身设计成游戏体验的核心，而非障碍**。

普通学习系统的问题：失败 = 挫败感 = 想逃避。

Math Souls 的重新定义：
- 失败 = 战斗日志（有价值的信息）
- 卡住 = 正在和Boss战斗（值得骄傲的状态）
- 解出来 = 击杀怪物（有仪式感的成就）

### 核心映射关系

| 魂类游戏元素 | 数学学习映射 | 设计意图 |
|---|---|---|
| 怪物 / Boss | 每一道数学题 | 题目不是评判工具，是值得征服的对手 |
| 死亡 / 失败 | 每次解题失败 | 失败记录在怪物页面，成为可回溯的信息 |
| 武器 / 装备 | 数学笔记、方法论 | 每次击杀可以获得/总结新武器 |
| 装备合成 | 手动合并两篇笔记 | 从多道题中抽象出更高层的方法 |
| 篝火 | （未来）阶段性总结 | 回顾、整理、从这里继续出发 |
| 血迹 / 残影 | （未来）历史失败记录 | 提醒自己的常见陷阱 |
| NG+ | （未来）同题用更高方法重做 | 螺旋上升，同一问题多维度理解 |
| 关卡闭环 | 数学各领域的知识图谱 | 拓扑和数分之间的暗门 |
| 开放世界 | 自由探索，不强制顺序 | 数学本来就是网状的，不是线性的 |

### 击杀哲学

**绝不看标准答案**是这套系统的灵魂。三种击杀方式对应不同的诚实程度：

- **⚔ 独立击杀**：完全自己解出，全额 XP，最高荣耀
- **💡 提示击杀**：看了提示但自己完成推导，50% XP，受援但有效
- **☠ 外挂击杀**：看了答案，0 XP，获得「诅咒」标记，下次必须重刷

外挂击杀不是惩罚，是如实记录。诅咒状态提醒你这道题还没真正征服。

## 文件结构

```
math-souls/
├── index.html      # 主入口，所有CSS样式，HTML结构，初始化逻辑
├── data.js         # 数据层：localStorage读写，所有业务数据操作
├── game.js         # 游戏引擎：Three.js场景、角色移动、怪物生成、巡逻AI
├── ui.js           # UI层：战斗面板、武器库、创建怪物、HUD更新
├── README.md       # 本文件
└── DESIGN.md       # 早期设计草稿（可参考历史思路）
```

### 文件职责边界

- `data.js` 只管数据，不碰 DOM，不碰 Three.js
- `game.js` 只管 3D 场景，通过回调通知外部（`onInteractCb`）
- `ui.js` 只管界面交互，调用 `DB.*` 读写数据，调用 `Game.*` 刷新场景
- `index.html` 负责把三者串联起来，处理键盘事件，初始化

### JS 加载顺序（不能改变）

```html
<script src="data.js"></script>   <!-- 1. 先加载数据层 -->
<script src="game.js"></script>   <!-- 2. 再加载游戏引擎 -->
<script src="ui.js"></script>     <!-- 3. 最后加载UI层 -->
```

三个文件暴露的全局对象：`DB`、`Game`、`UI`

## 快速开始

### 运行方式

直接用浏览器打开 `index.html` 即可。如果遇到字体加载问题（需要联网），或未来有跨域资源，用本地服务器：

```bash
# Python 方式
python -m http.server 8080
# 然后访问 http://localhost:8080

# 或用 VS Code 的 Live Server 插件
```

### 基本操作

| 操作 | 说明 |
|---|---|
| `W A S D` | 移动角色 |
| `E` | 靠近怪物时交互（进入战斗面板） |
| `ESC` | 关闭战斗面板 |
| 顶部「⚔ 召唤怪物」 | 创建新题目（怪物） |
| 顶部「📖 武器库」 | 查看/合成武器（笔记） |
| 顶部「💾 导出」 | 导出全部数据为 JSON 文件 |
| 顶部「📂 导入」 | 从 JSON 文件恢复数据 |

### 典型使用流程

1. 遇到一道不会的题 → 点「召唤怪物」，填入题目 LaTeX、难度、所属领域
2. 开始思考，失败了 → 进入怪物，点「战斗日志」→「记录新的尝试」，写下思路和失败原因
3. 反复尝试，直到解出 → 点「独立击杀」，填写解题故事，获得 XP 和武器
4. 如果实在解不出看了提示 → 「提示击杀」，50% XP
5. 武器（笔记）积累多了 → 打开「武器库」，把两件相关的合成为更高层的方法

## 游戏系统详解

### 世界地图

三块低多边形浮空大陆，用桥梁相连，漂浮在虚空中。

| 区域 | 主题色 | 地面色 | 中心坐标 | 尺寸 |
|---|---|---|---|---|
| 近世代数 | 紫色 `#8B5CF6` | 深紫 `#0e0920` | (-42, 0) | 38×32 |
| 高等代数 | 琥珀 `#F59E0B` | 深褐 `#120e00` | (22, -18) | 36×30 |
| 数学分析 | 翠绿 `#10B981` | 深绿 `#001510` | (22, 22) | 36×30 |

坐标系：X 轴向右，Z 轴向下（Three.js 默认），Y 轴为高度。

桥梁连接：代数↔高数，高数↔数分。

### 相机系统

- 类型：`OrthographicCamera`（正交投影，无透视变形）
- 视角：俯视 45 度，类似星露谷物语
- 视锥大小：`frustum = 10`（单位），可调整来改变可见范围
- 相机跟随：每帧用 `lerp(current, target, 0.09)` 平滑跟随玩家
- 初始位置：`camera.position.set(0, 45, 34)`，`lookAt(0, 0, 0)`

**调整视野大小**：修改 `game.js` 顶部的 `const fr = 10`，数值越小视野越近。

### 角色系统

- 移动速度：`SPEED = 0.13`（每帧移动单位数）
- 初始位置：近世代数大陆中心 `(-42, 0)`
- 外观：金色圆柱身体 + 多面体头 + 紫色披风 + 发光光晕
- 走路动画：移动时 Y 轴轻微上下浮动 `sin(t*8)*0.04`
- 光晕呼吸：`intensity = 0.8 + sin(t*2)*0.3`

**调整移动速度**：修改 `game.js` 顶部的 `const SPEED = 0.13`。

### 怪物系统

#### 五种难度造型

| 难度 | 外观描述 | XP | 主体几何体 |
|---|---|---|---|
| ★☆☆☆☆ | 幽灵小球 + 两根手臂 | 50 | SphereGeometry |
| ★★☆☆☆ | 方块身体 + 方块头 + 两条腿 | 120 | BoxGeometry |
| ★★★☆☆ | 圆柱躯体 + 多面体头 + 肩甲 | 250 | CylinderGeometry + DodecahedronGeometry |
| ★★★★☆ | 八面体身 + 四面体头 + 四肢 | 500 | OctahedronGeometry + TetrahedronGeometry |
| ★★★★★ | 二十面体核心 + 锥形王冠 + 4颗轨道球 | 1000 | IcosahedronGeometry + 轨道动画 |

#### 行为模式

- **巡逻怪**（`patrol: true`）：在 `patrol_radius` 半径内随机游走，每 2-5 秒换一个目标点，速度 `0.04` 单位/帧
- **守卫怪**（`patrol: false`）：原地缓慢自转，`rotation.y = elapsed * 0.5`
- **交互距离**：`INTER_DIST = 5.5`，进入范围后显示 `[E] 挑战` 提示
- **已击杀的怪**：倾斜 `rotation.z = PI/2.1`，半透明（opacity 0.5），不可再次交互

#### 名字标牌

每只怪物头顶有一个 Canvas 绘制的 Sprite，显示怪物名（最多16字）和难度星级。颜色跟随所属领域的高亮色（`area.hi`）。

### 角色等级系统

| 等级 | 称号 | 所需累计 XP |
|---|---|---|
| 1 | 迷途的学者 | 0 |
| 5 | 初入江湖 | 500 |
| 10 | 定理猎手 | 2000 |
| 15 | 证明剑客 | 5000 |
| 20 | 证明大师 | 10000 |
| 30 | 数学领主 | 25000 |
| 40 | 无穷维行者 | 60000 |
| 50 | 无限维存在 | 120000 |

等级计算在 `DB.addXP()` 中完成，每次击杀后自动更新。等级只升不降。

**新增等级**：在 `data.js` 的 `LEVEL_TABLE` 数组末尾追加 `{lv, xp, title}` 即可。

### 武器系统

武器 = 你总结的数学笔记/方法论。击杀怪物时可以新建武器，也可以在武器库中手动创建。

#### 图标系统（可扩展）

当前只用 emoji，但数据结构预留了升级空间：

```javascript
icon: {
  type: 'emoji',   // 未来可改为 'svg' | 'url' | 'component'
  value: '⚔️'     // emoji字符，或SVG字符串，或图片URL
}
```

升级到 SVG 图标时，只需修改 `ui.js` 中渲染 `w.icon.value` 的地方，判断 `type` 分支处理即可，不影响已有数据。

#### 装备合成

在武器库弹窗中选择两件武器 → 填写合成武器名称和描述 → 点击合成。
合成后的武器会记录 `synthesized_from: [id1, id2]`，原武器保留不删除。

### 战斗面板

进入怪物后全屏覆盖，分左右两栏：

**左栏（285px 固定宽）**
- 怪物 3D 旋转预览（独立的 WebGLRenderer，关闭时销毁防止内存泄漏）
- 怪物名称、难度星级、领域徽章
- 怪物故事（lore）
- 状态芯片：未曾交手 / 战斗中（闪烁）/ 已击杀 / 外挂击杀（诅咒）
- 标签列表
- 撤退按钮

**右栏（三个 Tab）**

1. **题目**：KaTeX 渲染 LaTeX 公式，折叠式提示列表，击杀操作按钮
2. **战斗日志**：按时间倒序显示所有失败尝试，可记录新的尝试（思路+失败原因+使用武器）
3. **武器**：当前武器库的网格视图

### Minimap

右下角 130×130 的 Canvas 小地图，每帧用 `drawMinimap()` 重绘：
- 蓝色/绿色矩形：三块大陆的轮廓
- 红点：存活怪物位置
- 灰点：已击杀怪物位置
- 金色大点：玩家当前位置（始终在中心）
- 视图跟随玩家，`SCALE = 0.9`（缩放比例）

## 如何扩展

### 新增数学领域

1. 在 `game.js` 的 `AREAS` 数组中添加一条记录：

```javascript
{ id:'topology', name:'拓扑学', color:0xEC4899, hi:0xF9A8D4, ground:0x1a0010, cx:80, cz:0, w:36, d:30 }
```

2. 在 `index.html` 的 CSS 中添加徽章样式：

```css
.badge-topology { background:rgba(236,72,153,.2); color:#F9A8D4; border:1px solid rgba(236,72,153,.4); }
```

3. 在 `ui.js` 中所有 `domainMap` 对象里添加 `topology: '拓扑学'`。

4. 在 `game.js` 的 `_buildBridge()` 调用处，添加新大陆与已有大陆的桥梁。

### 新增怪物字段

在 `data.js` 的 `createMonster()` 函数中，直接在 `extensions: {}` 里临时存放，或正式添加到结构中。已有怪物不受影响（旧数据的新字段会是 `undefined`，用 `|| defaultValue` 处理即可）。

### 升级武器图标为 SVG

1. 修改创建武器时传入 `icon_type: 'svg'`，`icon_value: '<svg>...</svg>'`
2. 在 `ui.js` 中所有渲染 `w.icon.value` 的地方，加一个判断：

```javascript
function renderIcon(icon) {
  if(icon.type === 'svg') {
    return '<span style="display:inline-block;width:1.3rem;height:1.3rem">' + icon.value + '</span>';
  }
  return icon.value;  // emoji 直接输出
}
```

### 新增击杀方式

在 `data.js` 的 `killMonster()` 中，`mult` 的计算逻辑可以随意扩展：

```javascript
const mult = {
  'independent': 1.0,
  'hinted':      0.5,
  'cheated':     0.0,
  'ng_plus':     1.5   // 新增：NG+击杀给额外奖励
}[kd.method] ?? 0;
```

### 接入后端 API

`data.js` 目前全部使用 `localStorage`。未来替换为 API 调用时：

1. 保持所有函数签名不变（`DB.getMonsters()`、`DB.saveMonster()` 等）
2. 把函数体从同步改为 `async/await`
3. `game.js` 和 `ui.js` 中调用 `DB.*` 的地方相应加 `await`
4. 这样可以做到最小改动量

## 设计备忘

### 为什么不自动匹配难度？

数学本来就不会自动匹配难度——你今天翻开 Rudin，可能第一页就是精英怪。这种「无情的真实性」是这套系统的核心体验，不应该被游戏化掉。比如，数学分析一上来就是实数理论😒

### 为什么战斗系统映射成笔记而不是操作？

数学的战斗在大脑里，游戏无法复现推导过程。所以「战斗系统」被抽象为「你用了哪些方法/武器」的记录，而不是实时的操作交互。

### 为什么击杀结算是手动的？

自动化结算需要判断「是否解出」，而这个判断在数学里非常主观（看了几分钟提示算不算？在网上搜了一个引理算不算？）。把这个判断权交给用户自己，是对自己的诚实，也是这套系统能长期使用的关键。

### 关于「外挂击杀」的设计

外挂击杀不是惩罚机制，是一个**诚实记录工具**。给0 XP是因为看答案没有真正建立理解，但这件事本身不该被羞耻化——有时候你确实需要看答案才能理解这道题在干什么。诅咒标记的意义是：下次看到这道题时，提醒自己「这里还有欠债」。

### v0.1.1（2026-03-20）

#### Bug 修复

**1. 击杀音效与击杀图标（`ui.js`）**

三种击杀方式现在各有独立的 Web Audio 合成音效和全屏动画特效卡片：

| 方式 | 音效风格 | 图标 | 颜色 |
|---|---|---|---|
| 独立击杀 | 四音阶上升号角（胜利感） | ⚔ | `#00FF99` 绿 |
| 提示击杀 | 柔和三音渐进（折中感） | 💡 | `#FFD700` 金 |
| 外挂击杀 | 低沉锯齿不和谐音（压抑感） | ☠ | `#FF4466` 红 |

确认击杀后会弹出带缩放弹入动画的特效卡片，显示图标 + 击杀方式 + XP，1.6 秒后自动淡出。

**2. KaTeX 渲染修复（`ui.js`）**

- **支持 `$$...$$`、`\[...\]`、`$...$` 定界符**：渲染前自动剥离外层定界符，`$$` 和 `\[\]` 为 displayMode，`$` 为 inline。
- **中文字符自动转义**：将 LaTeX 中的中文字符自动包裹为 `\text{中文}`，消除 `unicodeTextInMathMode` 警告。
- 传入 `strict: false` 选项彻底静默非严格模式警告。

**3. `MeshLambertMaterial: flatShading` 报错（`ui.js`）**

怪物预览面板 `_buildPreviewBody` 中，将 `MeshLambertMaterial` 改为 `MeshPhongMaterial`，该材质支持 `flatShading`。

#### 功能变更

**4. 视角拉近（`game.js`）**

正交相机视锥 `fr` 从 `26` 改为 `10`，视角更近，世界细节更清晰。修改 `game.js` 顶部 `const fr = 10` 可自行调整。

**5. 已击杀怪物：隐藏名字、强制静止（`game.js`）**

- 已击杀的怪物不再显示名字标牌（Canvas Sprite），保持神秘感。
- 无论创建时是否开启巡逻，击杀后强制静止（`patrol: false`、`dead: true`），不再自转。

**6. 武器库重构为纯静态笔记索引（`ui.js`）**

武器库（`📖 笔记库`）不再提供动态添加/合成功能，改为从 `ui.js` 顶部的 `NOTES` 数组读取静态条目。

**添加笔记的方式**：直接编辑 `ui.js` 顶部的 `NOTES` 数组：

```javascript
var NOTES = [
  {
    icon:   '📐',                          // emoji 图标
    name:   'Cauchy-Schwarz 不等式',        // 笔记名称
    domain: 'analysis',                    // 'algebra' | 'calculus' | 'analysis'
    desc:   '|⟨u,v⟩|² ≤ ⟨u,u⟩·⟨v,v⟩',   // 一句话描述
    link:   '../notes/cauchy-schwarz.html' // 指向你的 HTML 笔记，留空则无跳转
  },
  // ... 更多条目
];
```

点击有 `link` 的卡片会在新标签页打开对应的 HTML 笔记文件。笔记库按领域（近世代数 / 高等数学 / 数学分析）自动分组显示。

> **关于 `file://` 安全警告**：`index.html:1 Unsafe attempt to load URL file:///...` 是浏览器对 `file://` 协议的同源限制，**不影响游戏任何功能**，仅在控制台出现。用本地服务器（`python -m http.server 8080`）访问可消除。
